# 第一阶段：搞清楚要学什么，为什么要学

## 第 1 篇：《量化交易到底是在做什么？我为什么想学这个》

### 核心认知

**量化交易 ≠ 炒股软件**

很多人以为量化就是：看一堆指标 → 软件自动买卖。这是错的。

量化交易的完整流程是：
```
提出假设（为什么这个规律存在？）
    ↓
历史数据验证（回测，看这个假设过去是否成立）
    ↓
模拟盘验证（上仿真账户，排除回测和实盘的差异）
    ↓
小资金实盘（真钱验证，控制风险）
    ↓
持续迭代（策略有生命周期，市场变了就要换）
```

每一步都需要动手，不存在"配置好就能躺赚"。

---

### 个人做量化最现实的方向

**商品期货 CTA 趋势跟踪**，原因：
- 数据相对易得（Tushare、聚宽等有期货历史数据）
- 逻辑清晰（趋势跟踪：涨了就买，跌了就卖）
- 市场参与者结构比股票简单，不那么"卷"
- 双向交易（可以做空），策略空间更大

**不建议一开始碰的方向**：股票高频、期权、跨市套利——这些对数据、基础设施、资金量要求都很高。

---

### 为什么选 WonderTrader，不是 backtrader / vnpy

| 框架 | 定位 | 适合场景 |
|------|------|---------|
| backtrader | Python 研究工具 | 快速验证想法，不适合实盘 |
| vnpy | 全功能但重 | 功能多但配置复杂，适合有基础的人 |
| WonderTrader | 生产级基础设施 | C++ 内核高性能，Python 封装易用，回测/实盘用同一套代码 |

选 WonderTrader 的核心原因：**学会了能直接用于实盘，不是学完还要换框架**。

---

### 先泼一盆冷水

个人做量化，大概率亏钱。原因：
1. 策略研发能力需要时间积累，新手策略大多经不起实盘检验
2. 资金量小时，手续费和滑点占收益的比例很高
3. 过拟合风险：在历史数据上调出来的好看曲线，在实盘上大概率失效

**这件事值得做的理由**：不是"肯定赚钱"，而是"掌握了一套完整的量化研究能力"——这个能力本身有价值，无论是自己交易还是做量化相关的工作。

### 本篇动手任务

在继续之前，确认以下几件事：

- [ ] 我有一台可以用的 Linux 机器（**Ubuntu 18.04 强烈推荐**，因为 gcc-8 在 18.04 的 apt 源里开箱即用；其他版本需要额外处理 gcc 版本问题）
- [ ] 我有基本的 Python 基础（会写函数、会用 pip 安装包，**Python 3.8 及以上**）
- [ ] 我了解期货的基本概念（合约、多空、保证金）——不懂的话先补一下，否则后面的策略逻辑会看不懂
- [ ] 准备好网络环境，后续需要从 GitHub / Gitee 克隆代码

**两个仓库的地址（放在同一父目录下）：**

```bash
# WonderTrader C++ 主仓库（国内用 Gitee 更快）
git clone https://gitee.com/wondertrader/wondertrader.git

# wtpy Python 封装层（同级目录）
git clone https://gitee.com/wondertrader/wtpy.git
```

克隆完成后目录结构应该是：
```
~/work/
├── wondertrader/    ← C++ 主仓库
└── wtpy/            ← Python 封装层
```

**这个同级关系很重要**：编译完成后，`copy_bins_linux.sh` 脚本默认把编译产物复制到 `../wtpy/wtpy/wrapper/linux`，也就是与 wondertrader 同级的 wtpy 目录里。
