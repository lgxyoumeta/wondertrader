## 第三阶段：读懂配置，搞懂数据，能自己准备回测

### 第 4 篇：《WonderTrader 回测需要哪些配置文件？每个字段是什么意思》
**学习目标**：能独立配置一次新策略的回测，不再靠复制粘贴

- configbt.yaml 完整字段解析：
  - `replayer` 节点：`cta_stime`/`cta_etime`（回测时间范围，格式 YYYYMMDDHHmm）、`mode`（csv 或 bin）、`path`（数据路径）、`fees`（佣金配置文件路径）、`tick`（是否开启 tick 级回测）

  - commodities.json：品种信息（exchg、product、name、pricetick、volscale 等字段）

  - contracts.json：合约列表（具体到每个到期月份的合约）
  - holidays.json：节假日列表（影响 K 线是否生成）
  - hots.json：主力合约换月规则（date/from/to/oldclose/newclose 字段）

## 第四阶段：读懂策略源码，理解框架怎么驱动策略

### 第 7 篇：《逐行读 WtStraDualThrust.cpp：框架是怎么调用策略的》
**学习目标**：看懂一个真实 C++ 策略的完整逻辑，建立对回调机制的直觉

- 文件位置：`src/WtCtaStraFact/WtStraDualThrust.cpp`，同时对照 `WtCtaStraFact/WtStraDualThrust.h`
- 四个回调函数的触发时机：
  - `on_init`：策略创建时触发一次，用来读取参数（`cta_param_str`/`cta_param_int`/`cta_param_dbl` API）
  - `on_session_begin`：每个交易日开盘前触发，DualThrust 在这里用历史高低价计算当天的上下轨
  - `on_schedule`：定时触发（由 configbt.yaml 里 `strategy.params.period` 决定），主逻辑在这里：取 K 线、判断突破、设置仓位
  - `on_tick`：每个 tick 到来时触发，DualThrust 不用这个（日内突破用 bar 就够了）
- 策略通过 `ICtaStraCtx` 接口和框架交互，所有 API 以 `stra_` 开头：
  - `stra_get_bars(code, period, count, isDay)`：取 K 线切片，返回 `WTSKlineSlice*`
  - `stra_set_position(code, qty)`：设置目标仓位（正数做多，负数做空，0平仓）
  - `stra_get_position(code)`：查当前仓位
  - `stra_get_price(code)`：查最新价
  - `stra_log_info(fmt, ...)`：打日志
- WTSKlineSlice 怎么用：`slice->at(idx)` 返回 `WTSBarStruct*`，再访问 `.open/.high/.low/.close/.vol` 等字段；`slice->maxprice(head, tail)` 取区间最高价

### 第 8 篇：《用 Python 写一个自己的 DualThrust 策略，逐行标注每一步在做什么》
**学习目标**：能独立用 Python 实现一个 CTA 策略并跑通回测

- wtpy 的策略基类：继承 `CtaTemplate`，实现 `on_init`/`on_session_begin`/`on_schedule`/`on_tick` 四个方法
- 完整 Python 版 DualThrust 代码逐行注释：
  - `on_init` 里用 `self.stra_get_bars()` 做数据预热，避免第一个 `on_schedule` 时数据不够
  - `on_session_begin` 里计算当天的 HH（过去N日最高价中的最高）、LC（过去N日收盘价中的最低）等，算出上下轨
  - `on_schedule` 里判断当前价格是否突破上轨/下轨，用 `self.stra_set_position()` 设置仓位
  - 注意 `self.stra_get_bars()` 返回的 K 线切片，用 `bars.high(i)` / `bars.low(i)` / `bars.close(i)` 访问（Python 侧和 C++ 侧接口略有不同，是方法调用而非属性）
- 把 configbt.yaml 的 module 改为 Python 策略的路径，跑回测，对比 C++ 版和 Python 版结果是否一致

### 第 9 篇：《给 DualThrust 加成交量过滤，回测结果变好了吗？》

### 第 11 篇：《回测很赚钱，实盘为什么会亏？理解过拟合、滑点和手续费》
**学习目标**：建立对回测局限性的清醒认知，不被好看的曲线骗到

- 过拟合的直觉理解：用参数热力图（k1 vs k2 不同组合的年化收益热图）看策略稳定性——如果只有某个点特别好，周围都差，说明这个参数是"调"出来的，不是"发现"出来的

## 第七阶段：接仿真实盘，打通完整链路

### 第 14 篇：《从回测到仿真：WtRunner 配置全解析，跑通 CTA 实盘链路》
**学习目标**：完成从回测到仿真实盘的关键一跳

- 实盘和回测配置文件的核心区别：

### 第 15 篇：《仿真盘跑了一周，回测信号和实盘信号对不上，我是怎么排查的》
**学习目标**：掌握实盘对比分析的方法，理解回测和实盘的必然差异

- 对比方法：在策略的 `on_schedule` 里用 `stra_log_info` 记录每次触发时的目标仓位、当前价格、上下轨数值，和回测日志（BtLogs/Strategy_{id}.log）同时间点逐行对比
- 常见差异来源及排查方法：
  - 触发时间不同：回测是 bar 准确关闭时触发，实盘是 bar 关闭后第一个 tick 到来才触发，可能差几百毫秒
  - 开盘第一根 bar 的数据不同：集合竞价价格是否纳入第一根 bar，取决于 sessions.json 里的开盘时间配置
  - 成交价不同：回测按 bar 开盘价或收盘价撮合（取决于 mocker 配置），实盘是实际市价成交
- 关键结论：一定程度的回测/实盘差异是正常的，目标是让逻辑信号一致，不要求每笔成交价完全相同

## 第八阶段：加上风控，让系统可以安全长期运行

### 第 16 篇：《WonderTrader 的"离合器"到底是什么意思？配置一套风控参数》
**学习目标**：理解风控机制，为策略配置适合的风控参数

- 离合器的比喻：踩离合器时发动机还在转（策略还在产生信号），但动力不传到轮子（仓位不执行）；和断路器的区别是软衰减而非硬切断
- WtSimpRiskMon 的工作流程：每隔 `calc_span` 秒计算一次"当前动态权益 / 当日最高权益"的比率，比率低于 `(1 - inner_day_fd/100)` 时，调用 `setVolScale(risk_scale)` 把全局仓位倍率压低；如果继续下跌可以调到 0
- config.yaml 里 env.riskmon 节点完整配置示例（真实字段名）：
  - `active: true`
  - `module: WtRiskMonFact`
  - `name: WtSimpRiskMon`
  - `inner_day_active: true`（开启日内风控）
  - `inner_day_fd: 20.0`（当日权益从高点回撤超过 20% 触发）
  - `multi_day_active: false`（暂不开启多日风控）
  - `calc_span: 10`（每10秒检测一次）
  - `risk_scale: 0.3`（触发后仓位缩减至三成）
  - `base_amount: 500000`（基准资金规模，单位元）
- 如何根据策略的历史最大回撤设定阈值：历史最大日内回撤 10%，风控阈值设 15%~20%，给正常波动留空间
- 验证风控是否生效：查看 Logs/Riskmon/Riskmon.log，每次检测都会有一行日志，触发时日志里的 scale 值会变成设定的 risk_scale

### 第 17 篇：《从仿真到实盘：怎么判断一个策略是否值得用真钱跑》
**学习目标**：建立从仿真到实盘的决策框架，不盲目上真钱

- 标准一：仿真信号和回测信号的一致性达到 90% 以上（按触发时间和方向，不苛求成交价）
- 标准二：仿真期间的实际损益曲线和回测曲线的走势大致吻合（允许有滑点和手续费的差异）
- 标准三：至少观察一个完整的季度（覆盖不同行情阶段：趋势、震荡、大涨大跌）
- 标准四：仿真期间没有出现过任何技术异常（断线重连后仓位正确、换月后策略继续正常运行）
- 实盘前的清单：风控参数设好了吗？最大允许亏损是多少？如果连续亏损超过这个数要怎么处理？

## 第九阶段：延伸——做自己的量化研究

### 第 18 篇：《从零实现一个均值回归策略：螺纹钢与铁矿石的价差交易》
**学习目标**：独立完成一个非标准示例策略的研发全流程，体验 SEL 引擎的多标的调度

- 假设来源：螺纹钢（rb）和铁矿石（i）的价格长期协整（原材料和产成品关系），价差偏离均值过大时倾向于回归
- SEL 引擎配置：configbt.yaml 里 `env.mocker` 改为 sel，strategies 改用 `sel` 节点，module 填 WtSelStraFact 或自己的策略工厂
- Python 策略实现：继承 `SelTemplate`，实现 `on_calculate` 方法（对应 SEL 引擎的 `cbCalc` 回调），在回调里计算价差的 N 日均值和标准差，偏离时对两个品种分别设置目标仓位
- 回测结果分析：这个策略在什么市场环境下会失效——单边趋势行情中协整关系破裂，价差持续走扩而不回归

### 第 19 篇：《怎么把每次研究变成一篇可复现的报告》
**学习目标**：养成量化研究的规范习惯，为长期积累打基础

- 一份好的研究记录应该包含：假设来源（为什么认为这个可能赚钱）、数据说明（品种/时间范围/频率/数据来源）、策略逻辑（伪代码或流程图，不是完整代码）、回测配置（参数/手续费/滑点）、回测结果（资金曲线图 + 关键指标表）、参数敏感性分析、结论（这个假设成立吗？在哪些条件下成立？）
- 工具链推荐：策略代码用 Git 管理，每次改动留清晰的 commit message；回测结果用 Pandas 整理成 DataFrame 保存为 csv；可视化用 matplotlib；研究笔记用 Obsidian（支持双链，方便建立不同策略和品种之间的关联）
- 失败的研究同样要记录：假设被证伪本身就是信息，以后不会再踩同一个坑

### 第 20 篇：《学完了能做什么？几个现实的方向和务实的建议》
**学习目标**：清醒认知，合理规划下一步

- 方向一：自己做量化交易。现实预期：初期大概率亏钱，把每次亏损当成学费，建立正确的研究流程比追求短期盈利更重要；相对可行的方向是商品期货 CTA 趋势跟踪，从小资金开始
- 方向二：做量化技术服务。需求真实存在（小型私募/主观交易者）；建议先建立技术影响力（持续输出内容）再接活，不要一开始就冷启动接陌生人的单
- 方向三：找量化相关的工作。WonderTrader 的使用经验 + 可以展示的回测研究报告，是比简历上写"了解量化"有说服力得多的东西
- 最重要的建议：不要停在"学会了"这一步。把学到的东西用起来——哪怕是跑一个仿真盘观察着，每周看一次日志，研究一下信号的逻辑——这才是真的在做量化
