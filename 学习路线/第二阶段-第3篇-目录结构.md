## 第 3 篇：《WonderTrader 的目录结构是什么意思？编译完了我得到了什么》

### 为什么要专门搞懂目录结构

第一次看到 wondertrader 和 wtpy 两个仓库，加在一起几十个目录，很容易不知道从哪入手。
这篇的目标是：**能在脑子里建立一张地图**，知道每个目录负责什么，以后改配置、找代码不再懵。

---

### 一、wondertrader 仓库结构

```
wondertrader/
├── src/               ← 全部 C++ 源码，编译的对象
├── dist/              ← 示例配置和运行时文件（含 Windows 编译产物）
├── docker/            ← Dockerfile，Linux 构建环境参考
├── scripts/           ← 辅助脚本
├── images/            ← 文档图片
├── copy_bins_linux.sh ← 编译后把 .so 复制到 wtpy 的脚本
├── copy_bins_win.bat  ← Windows 版对应脚本
├── CMakeLists.txt     ← （不存在于根目录，在 src/ 里）
└── README.md
```

### 二、src/ 目录：C++ 源码模块详解

src/ 下有 60+ 个子模块，按职责分组理解：

**基础工具层（不用改，但要知道存在）**
- `WTSUtils/`：基础工具类（字符串处理、文件操作等）
- `WTSTools/`：数据结构（K线切片、tick 数据等核心对象）
- `Includes/`：所有对外接口头文件（`ICtaStraCtx.h`、`ITraderApi.h` 等），**学接口必看**
- `Share/`：跨模块共享的基础类

**核心引擎层（理解架构的核心）**
- `WtCore/`：四个策略引擎（CTA/SEL/HFT）的 C++ 实现，`WtEngine.h` 是基类
- `WtBtCore/`：回测引擎，包含 `HisDataReplayer`（历史数据回放）和 `CtaMocker`（模拟撮合）
- `WtUftCore/`：UFT 极速引擎（纳秒级，C++ 专属）

**Python 接口层（连接 C++ 和 Python 的桥梁）**
- `WtPorter/`：实盘用的 Python 接口导出（生成 `WtPorter.so`），`WtPorter.cpp` 里是所有导出函数
- `WtBtPorter/`：回测用的 Python 接口导出（生成 `WtBtPorter.so`）
- `WtDtPorter/`：数据采集用的 Python 接口（生成 `WtDtPorter.so`）

**数据链路层**
- `WtDtCore/`：数据采集核心逻辑
- `WtDataStorage/`：历史数据存储模块（bin 格式读写）
- `WtDtHelper/`：数据工具（csv ↔ bin 转换，`trans_csv_bars` 就在这里）

**行情解析器（Parser，接行情源的插件）**
- `ParserCTP/`：对接 CTP 期货行情（最常用）
- `ParserCTPMini/`：CTP Mini 版
- `ParserXTP/`：对接 XTP 行情
- `ParserUDP/`：UDP 广播行情接收（QuoteFactory → WtRunner 的内部广播）
- `ParserShm/`：共享内存行情

**交易接口（Trader，对接交易通道的插件）**
- `TraderCTP/`：对接 CTP 期货交易（最常用，SimNow 就是这个）
- `TraderCTPMini/`：CTP Mini 版
- `TraderXTP/`：对接 XTP 交易
- `TraderMocker/`：回测专用模拟交易（不实际下单）

**策略工厂示例（学策略接口的入口）**
- `WtCtaStraFact/`：CTA 策略工厂，含 `WtStraDualThrust.cpp`（**第一个要读的策略源码**）
- `WtHftStraFact/`：HFT 策略工厂示例
- `WtSelStraFact/`：SEL 多标的策略工厂示例
- `WtUftStraFact/`：UFT 策略工厂示例

**风控与执行**
- `WtRiskMonFact/`：风控监控插件（"离合器"机制在这里实现）
- `WtExeFact/`：执行算法插件（TWAP、最小冲击等）

**运行器（Runner，把各模块组装成完整进程）**
- `WtRunner/`：实盘运行器（生成 `WtRunner` 可执行文件）
- `WtBtRunner/`：回测运行器（C++ 版，对应 dist/WtBtRunner/）
- `QuoteFactory/`：行情采集服务（独立运行，落地历史数据）

### 三、dist/ 目录：示例配置和运行时文件

dist/ 是给 C++ 版运行器（WtBtRunner、WtRunnerCta 等）配套使用的，Python 用户（wtpy）参考其配置文件格式即可。

```
dist/
├── common/              ← 基础配置，所有运行器共用
│   ├── commodities.json ← 品种信息（pricetick、volscale 等）
│   ├── sessions.json    ← 交易时段（每个品种的开收盘时间、夜盘）
│   ├── contracts.json   ← 具体合约列表
│   ├── holidays.json    ← 节假日
│   ├── hots.json        ← 主力合约换月规则（198KB，记录了大量历史换月数据）
│   └── fees.json        ← 手续费配置
├── storage/
│   ├── csv/             ← CSV 格式历史数据
│   │   └── CFFEX.IF.HOT_m5.csv   ← 沪深300主力5分钟K线（示例数据）
│   └── bin/ticks/       ← Tick 级二进制数据（.dsb 格式）
│       ├── CFFEX.IF.HOT_tick_20210104.dsb
│       ├── CFFEX.IF.HOT_tick_20210105.dsb
│       └── CFFEX.IF.HOT_tick_20210106.dsb
├── WtBtRunner/          ← C++ 回测运行器配置示例
│   ├── configbt.yaml    ← 回测主配置（和 wtpy 的 configbt.yaml 格式一致）
│   └── logcfgbt.yaml    ← 日志配置
├── WtRunnerCta/         ← CTA 实盘运行器配置示例
│   ├── config.yaml      ← 实盘主配置
│   ├── tdtraders.yaml   ← 交易通道配置
│   ├── tdparsers.yaml   ← 行情通道配置
│   ├── executer.yaml    ← 执行器配置
│   └── logcfg.yaml      ← 日志配置
├── WtRunnerHft/         ← HFT 实盘运行器配置示例
├── WtRunnerUft/         ← UFT 实盘运行器配置示例
├── LoaderRunner/        ← 合约信息从 CTP 加载的工具
└── QuoteFactory/        ← 行情采集服务配置示例
    └── dtcfg.yaml       ← QuoteFactory 配置（parsers + writer 节点）
```

**重要提示**：dist/ 里的 `.dll` 文件是 Windows 编译产物，Linux 下没有用。Linux 上对应的是 `.so` 文件，来自你自己编译或 wtpy 包携带的版本。

### 四、wtpy 仓库结构

```
wtpy/
├── wtpy/                ← 核心 Python 包
│   ├── __init__.py      ← 导出 WtBtEngine、WtEngine 等主要类

│   ├── WtEngine.py      ← 实盘引擎的 Python 封装
```

### 五、编译后我实际得到了什么

执行完 `make -j4` + `copy_bins_linux.sh` 后，变化在这里：

```
wtpy/wtpy/wrapper/linux/ 新增了：
├── WtPorter.so          ← 实盘 Python 接口
├── WtBtPorter.so        ← 回测 Python 接口
├── WtDtPorter.so        ← 数据采集 Python 接口
└── Loader/              ← 合约加载器子目录
    └── *.so
```

当你 `import wtpy` 时，Python 会自动根据操作系统加载 `wrapper/linux/` 下对应的 `.so`，调用里面导出的 C 函数。这就是 wtpy 的工作原理：**Python 做逻辑编排，C++ 做高性能执行**。

### 六、一张图理解整体关系

```
你的策略代码 (Python)
    ↓ 调用
wtpy (CtaContext.py / WtBtEngine.py)
    ↓ ctypes 调用
WtBtPorter.so / WtPorter.so   ←── 你编译出来的
    ↓ C++ 内部调用
WtBtCore (回测引擎) / WtCore (实盘引擎)
    ↓
WtDataStorage (历史数据读取) + WtCtaStraFact (策略工厂)
    ↓
dist/storage/ 里的 .csv/.dsb 历史数据
```
